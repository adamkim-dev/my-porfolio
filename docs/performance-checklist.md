# Performance Checklist

This portfolio aims for a fast, accessible, and SEO-friendly experience.

## Performance considerations

- Use server-rendered page content for quick first load.
- Keep animations non-blocking and optimized.
- Use CSS placeholders for images instead of large assets.
- Avoid expensive JavaScript during page startup.

## SEO and accessibility

- Use descriptive metadata and page titles.
- Keep sections well-structured and semantically marked.
- Ensure color contrast is sufficient in both light and dark modes.
- Add `aria-label` to buttons where required.

## Audit checklist

- Run Lighthouse performance and accessibility audits.
- Confirm the page passes basic SEO checks.
- Verify the AI route responds quickly in the network panel.
- Ensure no unused CSS or JavaScript is blocking rendering.
